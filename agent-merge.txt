# Merge Agent 2 + Agent 3 in Reconciliation Stage

## Context

`backend/app/pipeline/reconciliation.py` currently runs two parallel per-cluster Haiku calls — Agent 2 (discrepancy scoring) and Agent 3 (canonicalization) — via two outer `asyncio.gather`s (reconciliation.py:486-495). For N clusters that's 2N requests, fanned out concurrently under a single shared `Semaphore(5)`. Both calls take the same input (`cluster_events_json` for the same cluster), so they double the request budget without doubling the information available to the model.

During the recent QI run we tripped a `RateLimitError` on cluster `c007` because the burst exceeded Anthropic's per-minute cap, and tenacity's 3-retry / max-10s backoff window was shorter than the 60s rate-limit refill. The graceful fallback (`_deterministic_score_fallback`) caught it, so output was preserved, but the underlying dual-call pattern is the root cause.

**Goal:** collapse Agent 2 + Agent 3 into a single Haiku call per cluster (`Agent R — review`) that returns both the discrepancy score *and* the canonical entry in one tool_use. This halves the request count, removes the second outer gather (so peak in-flight drops from 10 → 5), and slightly improves output coherence since both judgments are emitted from one model context.

The existing per-cluster fallbacks, semaphore gating, exception handling, and the public `reconcile()` signature stay untouched — Agent 1 (clustering) and Agent 4 (Sonnet critic) are not affected.

## Scope

1. Add merged prompt + tool to `backend/app/prompts.py`.
2. Replace `_score_cluster` + `_canonicalize_cluster` with a single `_review_cluster` in `backend/app/pipeline/reconciliation.py`.
3. Update the `reconcile()` orchestration to call `_review_cluster` once per cluster instead of fanning out two gathers.
4. Update the module docstring (`4-agent` → `3-agent`).
5. Remove now-unused imports of the old score/canon prompts and tools.
6. Optionally delete the old `RECONCILIATION_SCORE_*`, `RECONCILIATION_CANON_*`, `SCORE_DISCREPANCY_TOOL`, `CANONICALIZE_CLUSTER_TOOL` from `prompts.py` (nothing else references them).

**Out of scope:** Agent 1 clustering, Agent 4 critic, the `_gated`/`_CLUSTER_SEM` plumbing, tenacity retry config, prompt content of Agent 1/4, schemas, the public `reconcile()` signature.

RECONCILIATION_REVIEW_SYSTEM = """You are an EMS quality analyst. Given a cluster of events that all represent the same real-world clinical action, perform two tasks in one pass:

1. Score the cluster for discrepancies a QA reviewer should examine.
2. Produce a canonical description and classification for the timeline.

Discrepancy types:
- timing: agree on WHAT, but timestamps differ by more than 10 seconds across sources.
- clinical: disagree on WHAT — different drug doses, different interventions, conflicting vital values.
- phantom: only one source (typically the PCR) documents an action that should have been observable by another source.
- missing: only non-PCR sources document this action — the PCR has no counterpart.
- none: no discrepancy; sources are consistent.

Scoring rules:
- discrepancy_score in [0.0, 1.0]: 0.0 = none, 1.0 = severe.
  - timing: min(1.0, timestamp_spread_seconds / 120.0).
  - clinical: always >= 0.7.
  - phantom/missing: 0.4 routine; 0.8 for life-saving interventions (defibrillation, epinephrine, airway).
  - none: 0.0.
- discrepancy_reasoning: one neutral sentence citing specific evidence (timestamps or descriptions). No recommendations.

Canonicalization rules:
- canonical_description: concise, neutral, clinically precise; max 120 chars. Include doses if mentioned in any source. If sources conflict on a dose, describe the conflict briefly.
- event_type: most specific EventType that fits; prefer the PCR source's type when in doubt.
- canonical_timestamp_seconds: use the centroid provided. Do not recalculate.
- match_confidence: confidence in [0,1] that all cluster events describe the same action. 1.0 for a solo event or clearly identical multi-source cluster; 0.5–0.8 when descriptions are only approximately similar."""

RECONCILIATION_REVIEW_USER_TEMPLATE = """Review this cluster: score it for discrepancies AND produce its canonical entry.

Cluster ID: {cluster_id}
Centroid timestamp (seconds): {centroid_timestamp_seconds}

<cluster_events>
{cluster_events_json}
</cluster_events>

Use the review_cluster tool to return both outputs in one call."""

REVIEW_CLUSTER_TOOL: dict[str, Any] = {

### Step 2 — Replace `_score_cluster` + `_canonicalize_cluster` with `_review_cluster` (`backend/app/pipeline/reconciliation.py`)

Replace both functions (currently at reconciliation.py:327-404) with one:

```python
async def _review_cluster(
    cluster: EventCluster, by_id: dict[str, Event]
) -> tuple[ScoredCluster, DraftTimelineEntry]:
    """Agent R: one Haiku call that scores AND canonicalizes a cluster."""
    response = await claude_haiku(
        system=RECONCILIATION_REVIEW_SYSTEM,
        messages=[
            {
                "role": "user",
                "content": RECONCILIATION_REVIEW_USER_TEMPLATE.format(
                    cluster_id=cluster.cluster_id,
                    centroid_timestamp_seconds=cluster.centroid_timestamp_seconds,
                    cluster_events_json=_serialize_cluster_events(cluster, by_id),
                ),
            }
        ],
        tools=[REVIEW_CLUSTER_TOOL],
        max_tokens=1024,  # bumped from 512 to fit both outputs
    )

    tool_use = next(
        (b for b in response["content"] if b["type"] == "tool_use"), None
    )
    if tool_use is None:
        logger.warning(
            "Agent R (review_cluster) returned no tool_use for cluster %s — using deterministic fallbacks",
            cluster.cluster_id,
        )
        return (
            _deterministic_score_fallback(cluster, by_id),
            _first_event_fallback(cluster, by_id),
        )

    raw = tool_use["input"]
    scored = ScoredCluster(
        cluster_id=cluster.cluster_id,
        event_ids=cluster.event_ids,
        centroid_timestamp_seconds=cluster.centroid_timestamp_seconds,
        source_types=cluster.source_types,
        discrepancy_score=float(raw["discrepancy_score"]),
        discrepancy_type=DiscrepancyType(raw["discrepancy_type"]),
        discrepancy_reasoning=raw["discrepancy_reasoning"],
    )
    draft = DraftTimelineEntry(
        cluster_id=cluster.cluster_id,
        event_ids=cluster.event_ids,
        canonical_timestamp_seconds=float(raw["canonical_timestamp_seconds"]),
        event_type=EventType(raw["event_type"]),
        canonical_description=raw["canonical_description"],
        match_confidence=float(raw["match_confidence"]),
    )
    return scored, draft
```

Keep `_deterministic_score_fallback` and `_first_event_fallback` as-is — they are reused both for the no-tool-use branch above and for the per-cluster exception branch in `reconcile()`.

### Step 3 — Update the `reconcile()` orchestration (reconciliation.py:484-519)

Replace the dual `asyncio.gather` block with a single fan-out that returns tuples, then demux:

```python
    # Agent R — Per-cluster review (parallel, Haiku) — one call per cluster
    raw_results = await asyncio.gather(
        *[_gated(_review_cluster(c, by_id)) for c in clusters],
        return_exceptions=True,
    )

    scored_list: list[ScoredCluster] = []
    draft_list: list[DraftTimelineEntry] = []
    for cluster, result in zip(clusters, raw_results):
        if isinstance(result, Exception):
            logger.warning(
                "Agent R review failed for cluster %s — using deterministic fallbacks: %s",
                cluster.cluster_id,
                result,
            )
            scored_list.append(_deterministic_score_fallback(cluster, by_id))
            draft_list.append(_first_event_fallback(cluster, by_id))
        else:
            scored, draft = result
            scored_list.append(scored)
            draft_list.append(draft)
```

## Notes / risks

- The combined prompt is longer than either individual one. Token budget grows by ~40% per call but request count is halved, so net token usage drops slightly.
- `match_confidence` and `discrepancy_score` are now decided by the same model context — no risk of the two judgments contradicting each other on a clinical/timing edge case (a small quality win, not just a perf win).
- If the model occasionally truncates the combined output, increase `max_tokens` to 1536 before considering structural changes.
